# Requirements Document

## Introduction

The Maxi80Lambda REST API currently exposes a `/search` endpoint that proxies Apple Music search requests. Since the IcecastMetadataCollector Lambda now handles Apple Music search internally and stores artwork in S3, the `/search` endpoint is no longer needed. This feature replaces it with a `/artwork` GET endpoint that checks artwork existence in S3 via HeadObject and returns a pre-signed S3 URL for the artwork image. The S3 bucket remains private; the pre-signed URL grants time-limited read access to the specific object. The Lambda initialization is simplified by removing the Apple Music HTTP client and auth provider dependencies that were only needed for the search action.

## Glossary

- **Maxi80Lambda**: The AWS Lambda function that serves the Maxi80 REST API through API Gateway.
- **Router**: The component inside Maxi80Lambda that matches incoming API Gateway requests to registered Action handlers by endpoint path and HTTP method.
- **ArtworkAction**: The new Action handler that verifies artwork existence in S3 via HeadObject, generates a pre-signed S3 URL for the artwork, and returns it as a JSON response.
- **StationAction**: The existing Action handler that returns station information as JSON. Unchanged by this feature.
- **S3_Client**: The AWS SDK S3 client used to verify artwork existence in the S3 bucket via HeadObject calls and to generate pre-signed URLs for artwork objects.
- **Pre_Signed_URL**: A time-limited URL generated by the AWS SDK that grants temporary read access to a specific private S3 object without requiring the bucket to be public.
- **API_Gateway**: The AWS API Gateway that fronts the Maxi80Lambda, handling access token authentication and rate control.
- **IcecastMetadataCollector**: A separate Lambda that collects Icecast stream metadata, searches Apple Music, and stores artwork in S3 at the key pattern `{prefix}/{artist}/{title}/artwork.jpg`.


## Requirements

### Requirement 1: Remove the Search Endpoint

**User Story:** As a maintainer, I want to remove the `/search` endpoint from the Maxi80Lambda REST API, so that the API no longer exposes an unused Apple Music search proxy.

#### Acceptance Criteria

1. THE Maxi80Lambda SHALL NOT register a SearchAction in the Router actions list.
2. WHEN a request is received for the `/search` path, THE Router SHALL return a path-not-found error with HTTP status 404.
3. THE Maxi80Endpoint enum SHALL replace the `search` case with an `artwork` case mapped to the path `/artwork`.

### Requirement 2: Add the Artwork Endpoint

**User Story:** As an API consumer, I want a GET `/artwork` endpoint that returns a pre-signed S3 URL for the artwork of a given artist and title, so that I can fetch and display album artwork for the currently playing song.

#### Acceptance Criteria

1. WHEN a GET request is received at `/artwork` with `artist` and `title` query parameters, THE ArtworkAction SHALL verify the artwork exists in S3 at the key `{prefix}/{artist}/{title}/artwork.jpg` using a HeadObject call.
2. WHEN the artwork object exists in S3, THE ArtworkAction SHALL generate a pre-signed S3 URL for the artwork object and return a JSON response with HTTP status 200 containing the pre-signed URL.
3. WHEN the artwork object does not exist in S3, THE ArtworkAction SHALL return HTTP status 204 No Content with no response body.
4. IF the `artist` query parameter is missing from the request, THEN THE ArtworkAction SHALL return an error indicating the missing parameter.
5. IF the `title` query parameter is missing from the request, THEN THE ArtworkAction SHALL return an error indicating the missing parameter.

### Requirement 3: Return a JSON Response with Pre-Signed S3 URL

**User Story:** As an API consumer, I want the artwork endpoint to return a JSON object containing a pre-signed S3 URL with a configurable expiration, so that I can fetch the image directly from S3 with time-limited access.

#### Acceptance Criteria

1. WHEN the ArtworkAction returns a successful response, THE Maxi80Lambda SHALL set the response `content-type` header to `application/json`.
2. WHEN the ArtworkAction returns a successful response, THE Maxi80Lambda SHALL return a JSON body containing a `url` field with the pre-signed S3 URL for the artwork.
3. THE ArtworkAction SHALL generate the pre-signed S3 URL using the AWS SDK with the S3 key `{prefix}/{artist}/{title}/artwork.jpg` and the configured expiration duration.
4. THE pre-signed S3 URL SHALL have an expiration duration read from the `URL_EXPIRATION` environment variable, defaulting to a sensible value when the variable is not set.
5. WHEN the ArtworkAction returns a 204 response, THE Maxi80Lambda SHALL return no response body and no `content-type` header.

### Requirement 4: Simplify Lambda Initialization and Configure S3 Access

**User Story:** As a maintainer, I want the Maxi80Lambda initialization to no longer require Apple Music credentials or an HTTP client and to be configured for S3 pre-signed URL generation, so that the Lambda starts faster, has fewer dependencies, and can serve artwork URLs.

#### Acceptance Criteria

1. THE Maxi80Lambda initializer SHALL NOT create an HTTPClientProtocol instance for Apple Music API calls.
2. THE Maxi80Lambda initializer SHALL NOT retrieve Apple Music secrets from SecretsManager.
3. THE Maxi80Lambda initializer SHALL NOT create an AppleMusicAuthProvider.
4. THE Maxi80Lambda initializer SHALL accept an S3 client (or configuration) to pass to the ArtworkAction for HeadObject checks and pre-signed URL generation.
5. THE Maxi80Lambda initializer SHALL read the S3 bucket name and key prefix from environment variables.
6. THE Maxi80Lambda initializer SHALL read the pre-signed URL expiration duration from the `URL_EXPIRATION` environment variable, with a sensible default when the variable is not set.

### Requirement 5: Preserve Existing Behavior

**User Story:** As an API consumer, I want the station endpoint, API Gateway authentication, and rate control to remain unchanged, so that existing integrations continue to work.

#### Acceptance Criteria

1. THE StationAction SHALL continue to handle GET requests at `/station` and return station information as JSON.
2. THE API_Gateway access token authentication SHALL remain unchanged.
3. THE API_Gateway rate control configuration SHALL remain unchanged.
4. THE Router SHALL continue to return a path-not-found error for unrecognized paths.
5. THE Router SHALL continue to return a method-not-allowed error for recognized paths with unsupported HTTP methods.

### Requirement 6: Handle S3 Existence Check Errors

**User Story:** As an API consumer, I want the artwork endpoint to handle S3 errors gracefully, so that I receive meaningful HTTP error responses instead of unhandled failures.

#### Acceptance Criteria

1. IF an S3 HeadObject call fails for a reason other than the object not existing, THEN THE ArtworkAction SHALL return HTTP status 500 with an error description.

### Requirement 7: Update SAM Template for S3 Access and Remove SecretsManager

**User Story:** As a maintainer, I want the SAM template to grant the Maxi80Lambda S3 read permissions and remove the SecretsManager policy, so that the Lambda can verify artwork existence and generate pre-signed URLs without retaining unused secret access.

#### Acceptance Criteria

1. THE template.yaml SHALL remove the SecretsManager IAM policy from the Maxi80Lambda function only; the IcecastMetadataCollector function SecretsManager policy SHALL remain unchanged.
2. THE template.yaml SHALL add an IAM policy granting the Maxi80Lambda function `s3:GetObject` and `s3:HeadObject` permissions on the MetadataBucket contents.
3. THE template.yaml SHALL add the `S3_BUCKET` environment variable to the Maxi80Lambda function, referencing the MetadataBucket parameter.
4. THE template.yaml SHALL add the `KEY_PREFIX` environment variable to the Maxi80Lambda function.
5. THE template.yaml SHALL add the `URL_EXPIRATION` environment variable to the Maxi80Lambda function.
6. THE template.yaml SHALL remove the `SECRETS` environment variable from the Maxi80Lambda function only; the IcecastMetadataCollector function `SECRETS` environment variable SHALL remain unchanged.
7. THE template.yaml SHALL NOT remove or modify any externally-created SecretsManager secret resources referenced by the IcecastMetadataCollector function.
